# vigill357.github.io
# 无限理想乡 · 创作宇宙

> 一个用于展示原创小说、衍生故事与世界观设定的静态网站，部署于 GitHub Pages。

---

## 项目简介

本仓库是 **「无限理想乡」创作宇宙** 的官方展示站点，包含主线小说目录、衍生故事子页面、插画图鉴与世界观数据库四大板块。网站为纯静态前端（HTML + CSS + JS），无需后端，所有内容数据通过本地 JSON / txt 文件驱动，在 GitHub Pages 上开箱即用。

## 各文件详细说明

### `index.html` — 网站主页面

网站的主 HTML 入口，承载所有板块的结构与布局：

| 区域 | 说明 |
|------|------|
| 几何光标 | 由两个 SVG 组成的三脚架造型自定义光标，随鼠标移动 |
| 导航栏 `<nav>` | 固定顶部，包含站名、锚点链接、连载状态指示灯 |
| Hero 首屏 | 全屏封面，含标题、简介、快速入口按钮、实时统计信息面板 |
| §01 主线小说 | 展示主线长篇信息与最新章节列表（由 loader.js 动态渲染） |
| §02 衍生故事 | 三列卡片，展示最新三篇衍生故事（按日期降序）；编号 `02` 下方有「更多」入口链接至 `stories.html` |
| §03 插画图鉴 | 四列网格，展示角色/场景/概念插画占位框 |
| §04 世界观与设定 | 人物、地理、时间线、规则四个入口卡片 |
| 页脚 `<footer>` | 版权信息与最近更新日期 |

按依赖顺序引入三个 JS 脚本：`cursor.js` → `loader.js` → `app.js`。

### `novel.html` — 主线小说目录页

独立的完整章节列表页面，展示小说简介、统计数据与全部章节卡片。导航栏「衍生故事」指向 `stories.html`。

### `stories.html` — 衍生故事子页面「浅渊残响」

独立子页，专门展示所有衍生故事，由 `js/stories.js` 驱动：

| 区域 | 说明 |
|------|------|
| Hero 区 | 标签行「SIDE STORIES · 衍生故事」、主标题「浅渊**残响**」（青色高亮）、一句话描述 |
| 排序栏 | 三个切换按钮：「名称 A→Z」/ 「最新更新」/ 「字数」；点击后卡片实时重排，序号随排序重新编号 |
| 统计栏 | 动态填入「故事总数 · 累计字数 · 最近更新」 |
| 卡片网格 | 两列网格，每张卡片含序号、标题、简介、字数/日期元数据、右下角装饰大数字；hover 有荧光蓝白（`#A8E8FF`）底线扫入动画 |

### `reader.html` — 通用阅读器（双路由）

通过 URL 参数 `type` 区分主线与番外，最大程度复用同一套渲染逻辑：

| 参数示例 | 行为 |
|----------|------|
| `reader.html?ch=ch1` | 读取主线第 1 章（默认主线模式） |
| `reader.html?type=main&ch=ch1` | 同上（显式指定） |
| `reader.html?type=side&ch=story-01` | 读取番外「story-01」，切换青色主题 |

**双主题切换：** `type=side` 时 JS 在 `<html>` 挂载 `data-theme="side"`，CSS 变量自动将金色系（`--gold`）替换为青色系（`#5BC8E8`），所有按钮、边框、进度条、分隔线等视觉组件同步切换，无需逐一修改。

**主线模式特性：**
- eyebrow 标签显示「CHAPTER LOG」
- STATUS 显示「连载中」
- 正文编号格式 `// CH · 001`
- 导航：「返回锚点」→ `novel.html`，「深度上升 / 下降」

**番外模式特性：**
- eyebrow 标签显示「SIDE STORY · LOG」
- STATUS 硬写「完结」
- 正文编号格式 `// SIDE · 001`
- 导航：「返回锚点」改为「浅渊残响」→ `stories.html`，「上一篇 / 下一篇」
- 上下篇 href 均携带 `&type=side`，保持主题一致

### `js/cursor.js` — 几何光标模块

封装为 IIFE 模块 `Cursor`，对外暴露 `init()` 方法：

- **即时跟随**：监听 `mousemove`，内部三脚架（`#cur`）直接跟随鼠标坐标。
- **延迟跟随**：通过 `requestAnimationFrame` 循环，以 0.15 的插值系数让外部三角环（`#cur-ring`）平滑滞后跟随，产生惯性拖尾效果。
- **悬停放大**：对 `a`、`button`、`.story-card` 等可交互元素切换 `.big` 类，触发旋转放大动画。
- **点击波纹**：监听全局 `click` 事件，在点击坐标创建 `.rpl` 元素，0.65 秒后自动移除。

---

### `js/loader.js` — 内容加载与数据同步模块

封装为 IIFE 模块 `Loader`，对外暴露 `init()` 方法。负责从 `data/` 目录异步拉取数据并渲染到首页：

| 核心功能 | 说明 |
|----------|------|
| 读取章节索引 | 拉取 `data/main_stories/index.json`，获取所有章节文件夹名列表 |
| 加载章节元数据 | 对每个文件夹异步拉取 `meta.json`，获取标题、日期等信息 |
| 累计字数统计 | 读取各章节 `.txt` 正文，统计字符数并汇总显示在小说统计栏 |
| 渲染章节列表 | 将章节条目动态插入主页侧栏（倒序，最新在顶） |
| 同步统计面板 | 将章节总数、最近更新日期填入 Hero 信息面板 |
| 渲染衍生故事卡片 | 读取 `data/side_stories/index.json`，并行拉取各故事 meta + 字数，按日期降序取前三渲染到首页 §02 |
| 渲染世界观统计 | 读取 `world.json` 中的 `world` 字段，将各项数量填入世界观卡片 |

所有网络请求均包含 `try/catch`，失败时静默降级，保持 HTML 中的占位内容不变。

---

### `js/stories.js` — 衍生故事子页面逻辑模块

封装为 IIFE 模块 `StoriesPage`，对外暴露 `init()` 方法，仅在 `stories.html` 中引入：

- **并行加载**：`Promise.all` 同时拉取所有故事的 `meta.json` + 正文 txt，运行时统计字数。
- **三种排序**：名称（`localeCompare zh-Hans-CN`）/ 最新更新（date 降序）/ 字数（降序），切换后卡片实时重排，序号重新编号。
- **统计栏**：动态填入故事总数、累计字数、最近更新日期。
- **缓存复用**：`allItems` 缓存拉取结果，排序时直接操作内存，不重复 fetch。

---

### `js/app.js` — 主入口脚本

在 `DOMContentLoaded` 事件后执行：

1. **导航平滑滚动**：拦截所有 `href` 以 `#` 开头的锚点链接，改用 `scrollIntoView({ behavior: 'smooth' })` 平滑滚动。
2. **模块初始化调度**：依序调用 `Cursor.init()` 与 `Loader.init()`，启动光标系统与数据加载。

### `js/markup.js` — 自定义标记语言解析器

封装为 IIFE 模块 `Markup`，对外暴露 `parse(rawText)` 方法，将带标记的纯文本转换为 HTML 字符串，供 `reader.html` 渲染章节正文。

样式通过 `<style id="mk-styles">` 动态注入 `<head>`，仅注入一次。所有文本在输出前经 `esc()` 函数转义（`&`、`<`、`>`、`"` → HTML 实体），防止注入攻击。

#### A · 行内样式（嵌入段落中）

| 语法 | 效果 | CSS 类 |
|------|------|--------|
| `**文字**` | 金色强调（加粗） | `.mk-em` |
| `*文字*` | 内心独白（斜体淡蓝） | `.mk-thought` |
| `{{文字}}` | 数据/变量（等宽青色背景） | `.mk-var` |

> 解析顺序：`**` 先于 `*` 先于 `{{}}`，顺序不可颠倒。

#### B · 整段块（独占一行）

| 语法 | 效果 | CSS 类 |
|------|------|--------|
| `[[文字]]` | 居中蓝字（关键台词 / 信号） | `.mk-center` |
| `> 文字` | 系统日志（等宽青色左边框） | `.mk-log` |
| `<< 文字 >>` | 通讯频道（等宽绿色左边框） | `.mk-comm` |
| `((文字))` | 叙事旁白（右对齐灰色斜体） | `.mk-aside` |
| `!![[文字]]!!` | 系统警告（红色等宽方框） | `.mk-warn` |
| `^^^ 文字 ^^^` | 广播公告（金边居中） | `.mk-broadcast` |
| `\|\|\| 文字 \|\|\|` | 档案引用块（边框容器） | `.mk-archive` |
| `!!! 文字 !!!` | 危险叙事块（红色左边框） | `.mk-danger` |

#### C · 结构标记

| 语法 | 效果 | CSS 类 |
|------|------|--------|
| `---` | 场景切换（菱形分隔线） | `.mk-break` |
| `=== 文字 ===` | 章内小节标题（带分割线） | `.mk-section` |
| `### 文字` | 时间戳 / 地点标注（等宽小字） | `.mk-timestamp` |
| 空行 | 段落间距（1 行 = 普通间距，≥2 行 = 加大间距） | `.mk-gap` / `.mk-gap-lg` |

#### 排版规格

| 属性 | 值 |
|------|----|
| 正文字体 | Noto Serif SC，17px，行高 2.15 |
| 等宽字体 | Share Tech Mono |
| 正文颜色 | `#C0D4E8`（浅蓝灰） |
| 金色强调 | `#C8A84E` |
| 青色数据 | `#A8E6F5` |
| 系统日志 | `#5BC8E8` |
| 通讯频道 | `#8BE8A0` |
| 系统警告 | `#E06060` |

  },

    { "id": "char-01", "type": "character | scene | concept", "label": "标题" }

    "timelineNodes": 0,

  }

### `data/main_stories/` — 主线章节数据目录

每章独立存放于以章节编号命名的文件夹中：

```
main_stories/
├── index.json          # 文件夹名列表，如 ["ch1", "ch2"]
└── ch1/
    ├── meta.json       # 章节元数据
    └── ch1.txt         # 章节正文（使用 Markup 语法）
```

**`meta.json` 字段说明：**

```jsonc
{
  "title":        "序· 第一章",       // 章节标题
  "sub_title":    "「副标题」",        // 章节副标题
  "date":         "2026-04-01",       // 更新日期
  "synopsis":     "章节摘要",          // 简短描述
  "arc":          "序章",             // 所属弧线名
  "illus":        "illus.jpg",        // 封面插图文件名（同目录下）
  "illusCaption": "图注文字",          // 插图说明
  "illusTag":     "标签1 · 标签2"     // 插图标签
}
```

### `data/side_stories/` — 衍生故事数据目录

每篇故事独立存放，命名规则为 `story-NN/`：

```
side_stories/
├── index.json          # 文件夹名列表，如 ["story-01", "story-02", "story-03"]
└── story-01/
    ├── meta.json       # 故事元数据
    └── story-01.txt    # 正文（使用 Markup 语法）
```

**`meta.json` 字段说明：**

```jsonc
{
  "title":    "故事标题",
  "synopsis": "一句话简介",
  "date":     "2026-xx-xx"
}
```

txt 文件名规则与文件夹名一致（`story-01/story-01.txt`），供 `loader.js` 与 `stories.js` 统一拉取字数。

## 快速上手：更新内容

| 操作 | 做法 |
|------|------|
| 新增主线章节 | 在 `data/main_stories/` 下创建新文件夹（如 `ch3/`），添加 `ch3.txt` 和 `meta.json`，在 `index.json` 数组中追加 `"ch3"` |
| 新增衍生故事 | 在 `data/side_stories/` 下创建文件夹（如 `story-04/`），添加 `story-04.txt` 和 `meta.json`，在 `index.json` 追加 `"story-04"` |
| 修改小说信息 | 编辑 `world.json` 中的 `novel.title`、`novel.synopsis`、`novel.status` |
| 更新世界观统计数量 | 修改 `world.json` 中的 `world.characters` / `locations` 等字段 |
| 新增插画 | 在 `world.json` 的 `gallery` 数组中添加条目 |
| 修改站名/简介 | 直接编辑 `index.html` 中对应的文本节点 |
| 调整配色 | 修改 `css/main.css` 模块 1 中的 CSS 变量值 |
| 调整阅读器排版 | 修改 `js/markup.js` 中的 `CSS` 常量 |
